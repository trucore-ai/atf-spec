# Security Policy

## Reporting a Vulnerability

Please report security vulnerabilities privately.

- Email: **security@trucore.xyz**
- Subject line: `SECURITY REPORT: <short summary>`
- Include:
  - affected component (spec/docs/verifier behavior)
  - reproduction steps (if applicable)
  - expected vs actual behavior
  - any suggested mitigation

We will acknowledge receipt and work with you on a coordinated disclosure timeline.

## Scope

This repository contains public documentation/specifications.
Reports are welcome for:
- spec ambiguities that could cause verification bypasses
- inconsistencies between spec and observed production behavior
- security-sensitive documentation errors

Do NOT include secrets or private keys in reports.

## Supported Versions

This repo tracks the current public spec. If a security-relevant issue is found:
- we will patch documentation/spec promptly
- we will note changes in the README "Spec status & versioning" section or a future spec versioning document
